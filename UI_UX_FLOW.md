## 3. 상호작용 흐름

### 3.1 카메라 → 자동 페이지 감지 → OCR → TTS

```
[카메라 뷰]
     │
     ├─ 실시간 프레임 캡처 (30fps)
     │
     └─ 이전 프레임과 차이 비교
         │
         ├─ 차이 < 임계값: 계속 모니터링
         │
         └─ 차이 ≥ 임계값: 페이지 변경 감지! ✓
             │
             └─→ [페이지 변경 알림 (진동 + 음향)]
                 │
                 └─→ [이미지 캡처 및 정규화]
                     │
                     └─→ [OCR 실행 (Tesseract.js)]
                         │
                         └─ 신뢰도 계산
                             │
                             ├─ 신뢰도 < 60%
                             │   └─→ [사용자에게 수동 캡처 요청]
                             │
                             └─ 신뢰도 ≥ 60%
                                 │
                                 └─→ [텍스트 추출 완료 ✓]
                                     │
                                     └─→ [IndexedDB에 저장]
                                         │
                                         └─→ [자동 재생 설정 확인]
                                             │
                                             ├─ 자동 재생 OFF
                                             │   └─→ [사용자가 수동으로 ▶ 누를 때까지 대기]
                                             │
                                             └─ 자동 재생 ON
                                                 │
                                                 └─→ [TTS 자동 재생 시작]
                                                     │
                                                     └─→ [음성 재생 진행률 표시]
                                                         │
                                                         └─→ [음성 재생 완료]
                                                             │
                                                             └─→ [다음 페이지 감지 대기]
```

### 3.2 책 목록 → 책 상세 → 텍스트 편집

                     │   └─→ [저장]

                     │       └─→ [IndexedDB 업데이트]
                     │           │
                     │           └─→ [저장 완료 알림]

| 컴포넌트 | 설명 | 상태 |
|---------|------|------|

## 5. 반응형 디자인

### 5.1 화면 크기별 레이아웃

**모바일 (< 600px)**
- 전체 너비 활용
- 스택 레이아웃 (세로)
- 터치 친화적 버튼 (최소 44px)

**태블릿 (600px - 960px)**
- 양쪽 마진 (16px)
- 2열 레이아웃 (필요시)
- 터치 친화적 버튼 (최소 48px)

**데스크톱 (> 960px)**
- 최대 너비 1200px, 중앙 정렬
- 3열 레이아웃 (가능시)
- 마우스 대응 호버 효과

### 5.2 터치 제스처

- **싱글 탭**: 버튼 활성화, 페이지 네비게이션
- **더블 탭**: 줌 인/아웃 (이미지 뷰)
- **스와이프 좌/우**: 이전/다음 페이지
- **스와이프 위/아래**: 스크롤
- **핀치**: 텍스트 크기 조절

---

## 6. 접근성 (Accessibility)

### 6.1 스크린 리더 지원

- 모든 버튼에 aria-label 추가
- 이미지에 alt 텍스트 제공
- 폼 요소에 label 연결
- 제목 계층 구조 (H1, H2, H3) 준수

### 6.2 색상 대비

- 텍스트 대비: 4.5:1 이상 (WCAG AA)
- 아이콘 대비: 3:1 이상

### 6.3 키보드 네비게이션

- Tab 키로 포커스 이동
- Enter 키로 버튼 활성화
- Escape 키로 다이얼로그 닫기
- 포커스 인디케이터 시각적 표시

## 7. 에러 처리 UI

### 7.1 에러 메시지 및 복구

| 상황 | 메시지 | 복구 방법 |
|------|--------|----------|
| 카메라 권한 거부 | "카메라 권한이 필요합니다. 설정에서 허용해주세요." | 설정 열기 버튼 |
| OCR 실패 | "텍스트 인식에 실패했습니다. 다시 시도해주세요." | 다시 시도 버튼 |
| 저장 실패 | "저장에 실패했습니다. 인터넷 연결을 확인해주세요." | 다시 시도 버튼 |
| 저장소 용량 부족 | "저장소 용량이 부족합니다. 캐시를 삭제해주세요." | 캐시 삭제 버튼 |
| 네트워크 오류 | "네트워크에 연결할 수 없습니다. 오프라인 모드입니다." | 재시도 버튼 |

### 7.2 로딩 상태

- 스켈레톤 로더 (목록, 상세)
- 프로그레스 바 (OCR, 저장)
- 로딩 스피너 (파일 업로드)
- 로딩 텍스트 ("처리 중...")

## 8. 애니메이션 및 전환

### 8.1 페이지 전환

- Fade In/Out (기본)
- Slide Up (모달, 바텀 시트)
- Slide Left/Right (페이지 네비게이션)

### 8.2 상호작용

- 버튼 Press: 0.2초 스케일 축소
- 텍스트 입력: 0.3초 하단선 애니메이션
- 토스트: 0.3초 Fade In, 0.5초 유지, 0.3초 Fade Out

### 8.3 피드백

- 성공 메시지: 녹색 체크마크 + 사운드
- 에러 메시지: 빨간색 느낌표 + 사운드
- 진동 피드백: 페이지 감지 시 (설정 가능)
